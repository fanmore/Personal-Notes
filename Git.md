## Git

分布式版本控制

## 原理

在本地电脑中处理文件，放置本地暂存区，然后推送到版本库

在本地工作目录和暂存区之间有个 add 命令

在暂存区和版本库之间有个 commit 命令

版本库的交流
  * 推送 push
  * 拉取 pull
  
## 命令

* git add：将本地文件增加到暂存区
* git commit：将暂存区的内容提交到本地仓库（本地分支，默认master）
* git push：将本地仓库（本地分支）的内容推送到远程仓库（远程分支）
* git pull：将远程仓库（远程分支）的内容拉去到本地仓库（本地分支）
